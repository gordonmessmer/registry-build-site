Three Tools, One Ecosystem

Each tool handles a specific part of the build pipeline, working together to automate the journey from source to deployment.

dist-git-manager

Automated Dist-Git Repository Maintenance

Monitors upstream releases and automatically maintains RPM spec files, patches, and supplementary build files. Manages SemVer-based release branches, detects new upstream versions, and keeps your dist-git repositories synchronized with upstream projects.

See Example: Distribution Build

Building packages from auto-maintained dist-git repositories:

# build-assist.yaml
base: fedora-39-x86_64

build:
  - type: dist-git
    url: https://src.fedoraproject.org/rpms/
    packages:
      - nginx:f39
      - postgresql:f39
      - redis:f39
      - nodejs:rawhide

# Creates a package repository for traditional deployment

source-git-manager

Source-Git Repository Mirroring

Maintains full mirrors of upstream git repositories and manages release branches containing RPM specs. Tracks upstream branches and automatically merges changes, enabling builds directly from source without a separate release process.

See Example: Development Build

Building directly from source repositories - no release process needed:

# build-assist.yaml
base: fedora-39-x86_64

build:
  - type: git
    url: https://github.com/myorg/
    packages:
      - webapp:main
      - webapp-api:main
      - webapp-auth:main
      - webapp-utils:main

# Builds from current development state
# Perfect for rapid iteration and testing

rpm-build-assist

RPM Package Building

Builds RPM packages from either dist-git or source-git repositories. Supports multiple deployment methods including traditional package repositories and portable software collections with rpath, enabling cross-distribution compatibility.

See Example: Portable Software Collection

Creating a portable software collection with rpath for cross-distribution compatibility:

# build-assist.yaml
base: fedora-39-x86_64

build:
  - type: git
    url: https://github.com/myorg/
    packages:
      - python312:main
      - python312-pip:main
      - python312-requests:main
      - python312-flask:main

install:
  type: software-collection
  collection: python312
  packages:
    - python312
    - python312-pip
    - python312-requests
    - python312-flask

# Installs to /opt/python312 with rpath
# Works across distributions with compatible libc

How It Works

A streamlined workflow that connects developers to users

1

Upstream Release Detection

Tools automatically monitor upstream projects for new releases using services like Anitya, ensuring you never miss an update.

2

Automated Repository Maintenance

dist-git-manager or source-git-manager updates specs, manages branches, and synchronizes changes without manual intervention.

3

Package Building

rpm-build-assist builds packages from your managed repositories, creating deployable artifacts with your chosen distribution method.

4

Deployment to Users

Deploy via package repositories or portable software collections. Users get bug fixes and features faster, with less overhead for maintainers.
